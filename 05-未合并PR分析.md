## 2. OPEN PR(3 个)

### 2.1 [#77399 add DispatchKeySet related API](https://github.com/PaddlePaddle/Paddle/pull/77399)(2026-01-17 创建)

**状态**:OPEN — 设计争议,review 中。

**改动**:`TensorBase.h`(+57 行) — 加 `_is_zerotensor` / `_set_zero` / `is_conj` / `_set_conj` / `is_neg` / `_set_neg` 接口。

**未合并原因**:PR 描述里营员主动提了一个开放问题:

> `key_set_` 属性在 `c10/core/TensorImpl.h` 中定义,**是否有必要在 `paddle/phi/api/include/compat/ATen/core/TensorBase.h` 下新增该属性**?

这是一个**架构层面的设计选择**:

- **方案 A**:在 `TensorBase` 上新增 `key_set_` 字段,与 PyTorch 内存布局一致 — 但 Paddle 内部并不使用 dispatch key 路由,此字段对内核没用
- **方案 B**:把 `_is_zerotensor` 等谓词都用**计算的方式**实现(例如 `_is_zerotensor() = (impl_->numel() == 0)`),不增字段 — 节省内存但语义略有偏差(PyTorch 中 `_is_zerotensor` 是显式 flag,可以被 `_set_zero(true)` 设)

review 一直在两个方案间讨论。同时 #78070 已经把 `DispatchKey{,Set}.h` / `intrusive_ptr.h` 完整落地(+1775 行),所以**这个 PR 的紧迫性下降**。

**对功能的影响**:6 个谓词接口暂未对外暴露,但下游(DeepEP / DeepGEMM / FlashMLA)没人用到,未阻塞任何编译。

### 2.3 [#78902 use thread-local stream in c10::cuda::getCurrentCUDAStream](https://github.com/PaddlePaddle/Paddle/pull/78902)(2026-05-07 创建)

**状态**:OPEN — 修复进行中,等待 review。

**改动**:`c10/cuda/CUDAStream.cpp`(+53/-19)、`CUDAStream.h`(+0/-2)、`test/cpp/compat/c10_Stream_test.cc`(+254 行)。

**改动背景**:

[#78652](https://github.com/PaddlePaddle/Paddle/pull/78652) 解决了"compat ↔ GPUContext stream 双向同步"问题,但**牺牲了 thread-local 语义** — 新线程从主线程继承 current stream(违反 PyTorch 约定);同时 `setCurrentCUDAStream` 内部触发 `cudaStreamDestroy` 旧 stream,如果旧 stream 来自 compat 的 `getStreamFromPool()`(compat 持有所有权),destroy 后 compat 池里的句柄失效,后续重用即 SegFault。

#### #78902 的修复

```cpp
// CUDAStream.cpp(简化)
thread_local std::optional<cudaStream_t> tls_current_stream;

CUDAStream getCurrentCUDAStream(DeviceIndex device_index) {
  if (tls_current_stream.has_value()) {
    return CUDAStream::wrap(*tls_current_stream, device_index);
  }
  // fallback 到 GPUContext stream(保持与 Paddle Python 一致)
  return CUDAStream::wrap(phi::GetMutableGPUContext()->stream(), device_index);
}

void setCurrentCUDAStream(CUDAStream stream) {
  tls_current_stream = stream.stream();
  // 不再触碰 GPUContext,避免 destroy 问题
}
```

#### 配套测试

`c10_Stream_test.cc`(+254 行)— 多线程并发 / pool stream 重发 / 死锁回归。

测试仓 [#60](https://github.com/PFCCLab/PaddleCppAPITest/pull/60) 的 `test/c10/cuda/CUDAStreamDeadlockAlignmentTest.cpp`(+206 行)是对照测试。

#### 状态

PR 在 2026-05-07 创建,截至 2026-05-29 仍在 review。等待导师 [@BingooYang](https://github.com/BingooYang) 确认 thread-local 与 GPUContext fallback 的优先级取舍。
