# 02 · 文档贡献

> 范围:PaddleCppAPITest 仓库 `doc/` 目录下的文档贡献。包括按 namespace 组织的 API 文档、5 个 Agent skill、覆盖率脚本。

## 1. 主题概述

测试仓的 `doc/` 目录不只是"API 说明书",而是**兼容层的工程档案**:

- **API 文档** — 每个 compat 接口的签名、行为、与 PyTorch 的对齐情况、Paddle 后端调用路径
- **mismatch_api_record.md** — 每个 namespace 下专门一份,记录已知不对齐的接口、原因、规避方案(详见 [`03-mismatch复现.md`](03-mismatch复现.md))
- **compat_arch.md** — 关键架构的 Mermaid 图(`allocator_compat_arch.md`、`data_ptr_compat_arch.md`、`storage_compat_arch.md`、`stream_compat_arch.md`)
- **Skill 文档** — 写在 `.claude/skills/` 与 `.github/skills/`,作为新营员和 LLM 助手的标准操作流程

```markdown
# Skill: Compat Doc Authoring

When writing compat layer documentation, follow this template:

## 1. Interface list
- List all exported symbols in `<namespace>/<file>.h`
- Mark alignment status: ✓ aligned, △ partial, ✗ not aligned

## 2. Behavior alignment
- For each interface, describe:
  - PyTorch signature (with link to upstream source)
  - Paddle compat signature (in our header)
  - Underlying Paddle backend call

## 3. Mermaid diagrams
- For architecture-level docs (*_compat_arch.md), include Mermaid sequence diagrams

## 4. Mismatch cross-references
- Link to `mismatch_api_record.md` in the same directory

## 5. Test coverage
- Reference test files in test/<namespace>/<topic>Test.cpp
```
